Mission Editor Setup

  1. Add a Mission Start trigger → DO SCRIPT FILE → select Splash_Damage_3.4.2.lua
  2. Add a second Mission Start trigger → DO SCRIPT FILE → select splash_damage_config.lua

The first trigger loads the script with its defaults. The second overrides any values you want to change. When you re-select the file in the trigger, the .miz bundles a copy at that point — re-select after making changes.

Core Explosion Behavior

The main splash damage engine. Wave explosions and larger explosions are the two primary mechanisms that make ordnance feel impactful.
SettingTypeDefaultDescription
enable_radio_menu RETtrue/falsefalseAdds F10 radio menu to tweak splash settings in-game without editing the script
wave_explosions RETtrue/falsetrueBlast wave: spawns secondary explosions on objects radiating outward from impact, scaled by object size and distance. The core feature.
larger_explosions RETtrue/falsetrueAdds an extra explosion at the weapon impact point using per-weapon values from the built-in explosion table
use_dynamic_blast_radius RETtrue/falsetrueCalculate blast radius from weapon explosive power. If false, uses the fixed blast_search_radius instead.
dynamic_blast_radius_modifier RETnumber2Multiplier applied to the dynamically calculated blast radius. Higher = bigger blast area. Retribution divides UI value by 100.
blast_search_radiusnumber90Fixed blast wave search radius in meters. Only used if use_dynamic_blast_radius is false.
overall_scaling RETnumber1Global multiplier on all explosive power in the script. Scales everything up or down. Retribution divides UI value by 100.
only_players_weaponstrue/falsefalseIf true, only tracks weapons launched by human players. AI weapons won't get splash damage.
blast_stuntrue/falsefalseNot yet implemented. Placeholder for future stun mechanic.
